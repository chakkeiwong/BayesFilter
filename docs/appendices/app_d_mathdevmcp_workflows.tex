\chapter{MathDevMCP Workflows}
\label{app:bf-mathdevmcp-workflows}

MathDevMCP is used as an audit assistant, not as an oracle.  Its results should
be recorded with their status and limitations.

Useful commands for this monograph include:
\begin{itemize}
  \item \texttt{search-latex} to find source labels and neighborhoods;
  \item \texttt{extract-latex-context} to record exact source provenance;
  \item \texttt{audit-derivation-label} for bounded symbolic checks;
  \item \texttt{audit-kalman-recursion} for AST-level implementation structure.
\end{itemize}

The current solve-form Kalman audit found the expected solve-oriented operation
graph but reported missing strict log-determinant and trace classification, as
well as missing explicit shape and covariance guards.  The correct response is
to document those gaps and use the MacroFinance parity tests as additional
evidence, not to relabel the audit as a proof.
